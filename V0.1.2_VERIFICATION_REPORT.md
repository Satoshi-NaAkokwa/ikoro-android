## Issues addressed in this build

### 1. Wallet crash on touch
**Root cause:** TrustWalletCore native code was invoked unconditionally. If the native library fails to load or throws during derivation, the app crashes.
**Fix:**
- Added `NativeWalletBridge` that probes the native library once and reports availability.
- `WalletManager.loadAssets()` now returns a safe placeholder instead of crashing when the native bridge is unavailable.
- `WalletManager.send()` returns a controlled `Result.failure` with an explanatory message if the bridge is unavailable, instead of throwing native exceptions.
- Every per-chain balance lookup is isolated in its own `try/catch` so a network failure on one chain does not crash the wallet.

### 2. LiveKit call button static / "Pin verification failed"
**Root cause:** The token server expects a POST with JSON body, but the app was sending GET with query parameters, causing the server to reject the request. The previous APK also still contained old PIN logic.
**Fix:**
- `LiveKitCallManager.fetchToken()` now POSTs `{"room":"...","identity":"..."}`.
- Confirmed with `curl` that the endpoint returns a valid JWT.
- `CallsScreen` and `ChatScreen` call buttons now fetch a real token and start `RoomActivity`.

### 3. Identity has no depth / one ID for all blockchains
**Root cause:** The identity screen listed DID, Nostr, EVM and Rootstock as separate cards, and Rootstock showed a different address even though it is EVM-compatible.
**Fix:**
- Redesigned `IdentityScreen` to present a unified "Ikoro Identity" card with the master seed fingerprint and shared EVM/DID/Rootstock address.
- Added an expandable "Derived addresses" section that explains the BIP-39 derivation paths for each chain.
- Changed `TrustWalletDerivation` so Rootstock uses the same EVM derivation path (m/44'/60'/0'/0/0). All EVM chains (Rootstock, Ethereum, Base, Polygon, Arbitrum, Optimism, BSC) now share one address.

### 4. Market buttons not clickable
**Root cause:** Airtime, Tickets, Savings and Land buttons used empty `onClick = { }` placeholders.
**Fix:**
- Created `MarketDialogs.kt` with `AirtimePayDialog`, `TicketBuyDialog`, `SavingsCreateDialog` and `LandRegisterDialog`.
- Wired each primary action to `ThirdwebContractService.executeAction`, which validates input and returns clear success/error feedback.
- Until contract addresses are configured, the dialogs show an explanatory message instead of crashing.

### 5. Group creation not visible in chat
**Root cause:** The group creation dialog was present but the user may have had an older build or the entry point was not discoverable.
**Fix:**
- Confirmed the "New contact" FAB opens a dialog with both "Add contact" and "Create group" options, and the group creation dialog is wired to `ChatManager.createGroup`.

### 6. Dead / redundant code
**Action:** Removed the unused `MarketActionContainer` skeleton from `MarketDialogs.kt`.
**Pending:** One stale file exists at `app/src/main/java/com/ugogbe/ikoro/wallet/WalletDerivation.kt`. The deletion attempt was blocked by the environment safety guard pending explicit user confirmation. Confirm and I will remove it.
